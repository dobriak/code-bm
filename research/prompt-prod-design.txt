## Setup
This is a personal web application that makes the process of managing your own music library, creating playlists and enjoying them simple and pleasant. It will use HTML5 and Web components to stream the music directly.
The music library exists in a single directory accessible to the application. The music there (mp3 or flac files) is organized in a hiearchy by Artist / Album / Song. The files are generally tagged well with the correct ID3 tags, including album cover pictures and genre.
There is also a directory of jingles and other sound files that would be inserted into the playlist (over quiet moments or between songs)
### Backend functionality
* Scan recursively the music library folder (given the path) to extract all ID3 tags and associate each music file with its tags in the database
* Asynchronously scan each file for additional metadata such as:
- quiet passages (timestamps) within the first minute and the last two minutes of any song longer than 4 minutes
- other audio characteristics such as key, timbre, danceability, chromatic mixing qualities that will be added in future versions
* Ability to perform CRUD on playlists
* Ability to manage live music streaming queue
### Frontend functionality
* Administrator functionality workflow
- Logs in with simple email/password authentication (from backend/.env), then is redirected to admin control center with the following functionality:
- all Raidio operational settings: directory path to music library, path to jingles directory, whether to play an auto-playlist or random songs when no user playlist is loaded
- summary of all genres, artists, albums, songs, total playtime   
- trigger manual scan or rescan of the music library and or the jingle directory
- CRUD on auto-playsts
- ability to manage directly the playlist queue be it user or auto-playlist
- setting allowing or disallowing automatic jingle insertion over quiet sections of songs, minimum quiet period can be adjusted (in seconds)
- ability to insert jingles over currently playing song
* User functionality
- users do not have to log in. They are assigned funny sounding names in the style of docker images, which are in the 'adjective_scientist' format, for example: dreamy_ritchie, cranky_heyrovsky, big_dijkstra.
- anyone can interact with the system and it is ok for playlists to compete for playing time. If more than one user playlists are loaded, play them in a round robin fashion
- users can view the album art of the currently playing song, and the artist-album-song name of 3 songs played before and 3 songs coming after the current one
- users can create playlists by dragging and dropping songs to a playlist. They have the ability to do fuzzy searches on all metadata about the songs, in particular the Artist / Album / Song name / Genre / Year / Duration.
- they can see visually if there are any quiet time spans on any song found via search or listing
- they can also insert jingles or other audio over the quiet passages or between the songs
### Player functionality
- pleasant design that focuses on the currently played song, displaying all its atributes with upcoming entries also visible
- all typical player controls
- dark / light theme toggle
- show only album art (full screen) toggle / untoggle
- remaining time of the currently playing song
- Web Audio API with minimal visualization (bars or waves) that can also be turned off via toggle
### Playlist creator functionality
- playlist name and separate input for notes, example: "night time playlist", "mellow music, mostly female vocals and electronic sound"
- two scrollable lists: one for browsing through the available songs, one to build the playlist via drag and drop from the existing song list
- search box on top of the available songs list that can filter by genre, artist, album, song name
- ability to browse the entire music library by genre, artist, album
- "feeling lucky" button that picks one random song and adds it to the playlist being created
- playlists can be saved and loaded as text files. They are generally stored in the current user's local storage
- user can browse, edit, or send the play queue their playlists
